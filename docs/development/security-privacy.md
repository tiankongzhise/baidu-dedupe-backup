# 安全与隐私开发要求

## 1. 文档目的

本文档定义 Baidu Dedupe Backup 首期开发中必须遵守的安全、隐私、授权、加密和误操作防护要求。它用于指导实现、测试和上线检查。

## 2. 安全原则

1. 默认开启加密备份。
2. 关闭加密必须有明确风险提示和二次确认。
3. 百度网盘授权仅用于备份相关操作。
4. 删除任务记录不等于删除百度网盘文件。
5. 解绑设备、解绑百度网盘、删除任务等高影响操作必须二次确认。
6. 公开仓库不得提交本地账号、授权凭据、密钥、真实测试文件或真实用户数据。

## 3. 账号安全

- 密码不得明文保存。
- 登录失败次数过多时，应触发安全验证或短暂限制。
- 访问令牌应设置有效期。
- 退出登录后，客户端应清理本地会话引用。
- 忘记密码或重置密码入口应验证用户身份。

## 4. 百度网盘授权安全

- 授权凭据不得暴露给 UI 层。
- 客户端不应长期保存未加密授权凭据。
- 服务端保存授权凭据时应使用安全存储或密钥托管机制。
- 授权状态失效时，任务必须暂停或中断，并引导重新授权。
- 解除绑定前必须提示未完成任务将无法继续。

## 5. 文件访问隐私

- 用户选择文件或文件夹后，产品才读取对应内容。
- 不应扫描用户未选择的系统目录。
- 不应在日志中记录完整敏感路径和文件内容。
- 本地路径用于用户可见展示和任务恢复时，应避免上传不必要的隐私信息。
- 测试和调试时应使用虚构路径和样例文件名。

## 6. 加密备份要求

### 6.1 默认行为

- 新任务默认开启加密。
- 设置页默认加密偏好建议保持开启。
- 任务和记录中必须展示“已加密”或“未加密”标识。

### 6.2 关闭加密

关闭加密前必须展示：

> 关闭加密后，备份内容将以未加密形式保存。请确认该任务不包含敏感资料。

确认按钮应使用明确文案，例如“确认关闭加密”。取消按钮应保留加密开启。

### 6.3 加密实现边界

- 具体加密算法不在 PRD 范围内，但实现必须选择成熟、公开、被广泛审查的方案。
- 不允许自行设计加密算法。
- 密钥管理方案需要单独评审。
- 发生加密失败时，任务不得继续上传未加密备份包。

## 7. 去重隐私要求

- 去重展示应解释结果，不展示复杂技术细节。
- 去重索引用于识别重复内容，不应反向暴露文件内容。
- 跨设备展示重复来源时，只展示设备名称、历史任务名称和必要文件名。
- 已解绑设备的历史记录可展示，但应明确设备已解绑。

## 8. 本地状态安全

本地可以保存：

- 设备本地标识。
- 登录态引用。
- 任务恢复点。
- 上传进度。

本地不应保存：

- 明文密码。
- 明文长期授权凭据。
- 不必要的文件内容副本。
- 未加密敏感日志。

## 9. 日志要求

日志可以记录：

- 任务 ID。
- 状态变化。
- 错误码。
- 耗时和进度统计。
- 匿名化设备类型。

日志不应记录：

- 明文密码。
- 授权令牌。
- 用户真实文件内容。
- 大量完整本地路径。
- 加密密钥。

## 10. 高影响操作确认

| 操作 | 是否二次确认 | 必须说明 |
| --- | --- | --- |
| 关闭加密 | 是 | 备份内容将以未加密形式保存 |
| 删除任务 | 是 | 任务记录消失，但云盘文件不自动删除 |
| 解绑当前设备 | 是 | 再次使用需要重新绑定 |
| 解绑非当前设备 | 是 | 历史记录保留，云盘文件不删除 |
| 解绑百度网盘 | 是 | 未完成任务无法继续，后续备份需重新授权 |

## 11. 上线前安全检查

- 注册、登录、退出登录流程不泄露敏感信息。
- 百度网盘授权凭据不出现在日志、前端状态和错误提示中。
- 关闭加密必须经过二次确认。
- 删除任务不默认删除百度网盘文件。
- 解绑设备和解绑百度网盘均有二次确认。
- 异常恢复不会重复上传已完成内容。
- 仓库不包含真实账号、密钥、授权信息或真实用户文件。
