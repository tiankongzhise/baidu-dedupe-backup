# 开发路线图

## 1. 文档目的

本文档将 PRD、功能规格和设计文档拆成可执行的版本计划，帮助后续研发确定里程碑、交付顺序和每个阶段的完成标准。

## 2. 版本策略

项目首期建议按三个版本推进：

1. **V1.0 基础可用版**：跑通单设备备份主链路和基础异常恢复。
2. **V1.1 多设备增强版**：补齐多设备管理、去重来源解释和通知中心。
3. **V1.2 体验优化版**：完善统计、批量管理、默认偏好和操作日志。

## 3. V1.0 基础可用版

### 3.1 目标

用户能完成注册登录、百度网盘授权、当前设备绑定、选择文件或文件夹、创建并完成备份任务，且任务默认加密、支持暂停恢复和基础异常恢复。

### 3.2 范围

- 注册、登录、退出登录。
- 百度网盘授权、授权状态查看、重新授权。
- 当前设备绑定。
- 文件或文件夹选择。
- 创建备份任务。
- 默认加密，可选择不加密。
- 关闭加密风险确认。
- 基础去重分析，可先支持同账号历史记录级别的重复识别。
- 任务开始、暂停、恢复、删除。
- 任务进度查看。
- 基础异常恢复。

### 3.3 不包含

- 完整多设备管理页。
- 通知中心完整能力。
- 批量任务管理。
- 复杂历史记录筛选。
- 定时自动备份。
- 恢复到本地。

### 3.4 完成标准

- 新用户可完成首次使用流程。
- 已注册用户可完成一次加密备份。
- 用户关闭加密时必须二次确认。
- 任务中断后能从任务列表继续。
- 删除任务不会默认删除百度网盘文件。

## 4. V1.1 多设备增强版

### 4.1 目标

强化多设备统一管理和去重解释，让用户知道重复项目来自哪台设备和哪个历史任务。

### 4.2 范围

- 设备列表。
- 当前设备标识。
- 设备重命名。
- 设备解绑。
- 查看设备备份概况。
- 多设备去重结果展示。
- 按设备查看备份记录。
- 异常通知中心。
- 更完整的历史记录筛选。

### 4.3 完成标准

- 用户可查看账号下所有设备。
- 当前设备有明确标识。
- 解绑设备前有二次确认。
- 已解绑设备的历史记录仍可查看。
- 去重详情可展示来源设备和历史任务。
- 通知中心可展示任务异常、授权失效和空间不足。

## 5. V1.2 体验优化版

### 5.1 目标

提升长期使用体验，增强统计、恢复引导、批量管理和默认偏好。

### 5.2 范围

- 更详细的节省空间统计。
- 更清晰的任务恢复引导。
- 批量任务管理。
- 默认备份偏好设置。
- 更完整的操作日志。
- 更完善的用户提示。

### 5.3 完成标准

- 首页展示累计跳过重复项目数和累计节省空间。
- 恢复任务时明确展示已完成内容和将继续内容。
- 用户可设置默认加密偏好，关闭时有风险确认。
- 操作日志可辅助排查任务失败原因。

## 6. 推荐开发里程碑

### 6.1 里程碑一：工程基础与领域模型

交付：

- 项目技术栈和仓库结构。
- 领域模型定义。
- 任务状态机。
- 统一错误码。
- 基础 Mock 数据。

完成标准：

- 状态机和错误码有单元测试。
- 主要实体字段与 `docs/development/data-model.md` 对齐。

### 6.2 里程碑二：账号、授权、设备闭环

交付：

- 注册、登录、退出登录。
- 百度网盘授权状态。
- 当前设备绑定。
- 首页基础状态。

完成标准：

- 未登录、未授权、未绑定设备的拦截逻辑正确。
- 用户能进入可创建任务的状态。

### 6.3 里程碑三：任务创建与去重分析

交付：

- 文件或文件夹选择。
- 待备份项目清单。
- 任务设置。
- 去重分析页。
- 确认创建任务。

完成标准：

- 用户能看到需备份、重复跳过和异常待确认结果。
- 关闭加密必须确认。

### 6.4 里程碑四：备份执行与进度

交付：

- 任务开始、暂停、恢复。
- 打包与加密。
- 百度网盘上传。
- 进度上报。
- 任务详情。

完成标准：

- 用户可完成一次真实或测试云盘备份。
- 任务详情展示完整进度和结果。

### 6.5 里程碑五：异常恢复与记录

交付：

- 异常中断识别。
- 恢复点。
- 继续备份。
- 备份记录。
- 删除任务确认。

完成标准：

- 断网、应用关闭、授权失效、空间不足至少有可验证处理路径。
- 删除任务不删除云盘文件。

### 6.6 里程碑六：多设备增强

交付：

- 设备管理页。
- 多设备去重来源。
- 备份记录筛选。
- 通知中心。

完成标准：

- 多设备历史备份可参与去重。
- 用户能看懂重复内容来自哪里。

## 7. 风险与依赖

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 百度网盘授权或上传限制 | 影响备份闭环 | 尽早做云盘适配器和 Mock |
| 大文件夹扫描耗时长 | 影响创建任务体验 | 提供扫描进度和取消能力 |
| 加密与恢复点设计复杂 | 影响恢复可靠性 | 单独评审加密与恢复策略 |
| 去重算法实现偏差 | 影响重复识别准确性 | 按 `dedupe-strategy.md` 实现文件级内容指纹去重，后续再评估块级增强 |
| 本地路径隐私 | 影响用户信任 | 最小化上传路径信息和日志记录 |
