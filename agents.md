# Agents.md

本文件用于沉淀项目中值得长期记住的知识、决策和协作约定，指导后续产品、设计、开发和 AI Agent 协作。

## 项目定位

项目名称：Baidu Dedupe Backup。

中文定位：多设备去重备份工具。

产品目标：帮助用户选择文件或文件夹后，默认加密打包备份到百度网盘，并通过多设备历史备份对比减少重复备份和云盘空间浪费。

当前仓库以产品规划文档为主，暂不包含技术实现。

现有核心文档：

- `README.md`：项目简介。

## 产品原则

1. 默认安全：备份任务默认加密，关闭加密必须有明确风险提示。
2. 用户可理解：去重、异常恢复、授权状态等能力必须用普通用户能理解的语言呈现。
3. 多设备统一：设备、任务、备份记录都应围绕同一用户账号统一管理。
4. 异常可恢复：断网、关机、应用关闭等中断场景不能让用户感觉任务丢失。
5. 删除要克制：删除任务记录不应默认删除百度网盘中已完成的备份文件。
6. 高影响操作需确认：解绑设备、解绑百度网盘、关闭加密、删除任务都需要二次确认。

## 已确定的产品范围

首期应覆盖：

- 注册、登录、退出登录。
- 百度网盘授权、重新授权、解除绑定。
- 当前设备绑定。
- 多设备管理。
- 创建备份任务。
- 文件或文件夹选择。
- 默认加密备份，可选择不加密。
- 多设备去重分析。
- 任务开始、暂停、恢复、异常中断恢复、删除。
- 备份进度和备份结果查看。

首期不覆盖：

- 百度网盘之外的其他云盘。
- 企业后台。
- 团队共享空间。
- 文件在线预览和在线编辑。
- 手机相册自动备份。
- 自动同步所有系统目录。

## 文档维护约定

1. 任何新增的重要产品决策，都应同步更新 PRD 或本文件。
2. 临时想法不要直接塞进本文件，只有确认会影响后续判断的内容才沉淀。
3. 本文件应保持简洁，优先记录稳定原则、已确认范围和重要约束。
4. 若 PRD 与本文件冲突，以最新明确的用户要求为准，并同步修正文档。

## 后续开发提醒

1. 开始技术实现前，需要先把 PRD 拆成可执行的版本计划和任务清单。
2. 涉及百度网盘授权、账号安全、备份加密时，要优先考虑用户风险提示和误操作防护。
3. 涉及多设备去重时，前台展示要解释结果，不展示复杂技术细节。
4. 涉及任务恢复时，页面上必须明确告诉用户哪些内容已完成、哪些内容将继续。
5. 公开仓库中不要提交本地账号、授权、密钥、测试文件或真实用户数据。
